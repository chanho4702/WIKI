# 협업 문서 플랫폼 로드맵 (Notion + Confluence 지향)

2026-07-17 기준. 사용자가 정의한 26개 기능 영역을 단계·분담·의존관계로 재정리한 문서다.
백엔드 계약 상세는 [wiki-service 요구사항](../backend/2026-07-17-wiki-service-requirements.md) 참조.

## 전역 결정 (2026-07-17 확정)

| 결정 | 내용 |
| --- | --- |
| 편집기 | WYSIWYG 블록 에디터 (TipTap 기반) |
| 저장 포맷 | **마크다운 문자열 유지** (`Page.body`) — 백엔드 계약·버전 diff·`[[링크]]` 파이프라인 무변경 |
| 편집 모드 | 컨플루언스식 보기/편집 분리 + 명시적 저장 (저장 1회 = 버전 1개 의미 유지) |

마크다운 저장 결정과 **충돌**하는 기능(토글, 콜아웃, 비디오, 임베드, 데이터베이스 뷰, 글자색, 밑줄 등)은
3단계에 배치하고 `⚠️ 저장 포맷 재논의` 플래그를 달았다. 해당 기능 착수 전에
확장 문법(:::) 또는 블록 JSON 전환을 다시 결정해야 한다.

## 범례

- ✅ 완료 (프론트, localStorage 목 스토어 기준)
- 🔨 진행 예정 (이번 웨이브)
- 🖥️ 프론트 주도 / 🗄️ 백엔드 선행 필요
- ⚠️ 저장 포맷 재논의 필요

## 비기능 요구사항 (백엔드 설계 입력)

조회 500ms 이하 · 자동 저장 1~3초(실시간 협업 도입 시) · 동시 편집 수백~수천 명 확장 ·
가용성 99.9% · 정기 백업 · TLS/암호화 저장/RBAC · 주요 작업 감사 로그 · 수평 확장.
→ wiki-service 및 인프라 설계 시 수용. 프론트 단독으로는 해당 없음.
