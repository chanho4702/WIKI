# 컨플루언스 충실 복제 — 글로벌 셸 + 디렉토리 테이블 설계

작성 2026-07-22. 근거: 사용자 제공 캡처 `사이드바 및 헤더 참고.png`(헤더+글로벌 사이드바+토글), `특정 스페이스 페이지.png`(스페이스 사이드바), `space 페이지.png`(디렉토리 테이블), home*.png. **캡처 = 스펙. 자체 판단(카드화 등) 금지, 그대로 복제.**

배경: 지금까지 화면마다 다른 셸(홈/디렉토리는 WikiTopBar만, 스페이스는 WikiLayout 사이드바)이라 **토글이 나타났다 사라지고 사이드바 구성이 화면마다 달라** 컨플루언스와 어긋났다. 정공법 = **하나의 글로벌 셸**로 통일.

## 1. 목표 구조 (캡처 그대로)

### 1.1 헤더 (모든 화면 상시)
좌→우: **사이드바 토글(⊞, 상시)** · 앱그리드(⋮⋮, 후속) · 로고(WIKI→/home) · **검색(중앙)** · 만들기 · (Rovo/알림/도움말은 후속 생략) · 다크모드 · 아바타.
- 핵심: **토글이 어느 화면에서든 항상** 헤더 좌상단에 있다(현재는 스페이스에서만 나옴 — 이게 "사라짐"의 원인).

### 1.2 좌측 사이드바 (상시, 토글로 접힘) — 3단
1. **글로벌 네비**(모든 화면 공통): 추천(→/home) · 최근(플라이아웃) · 별표 표시(플라이아웃) · 스페이스(→/spaces, 플라이아웃) · 앱. 현재 활성 항목 하이라이트.
2. **컨텍스트 섹션**(라우트별):
   - 홈·디렉토리: "별표 표시된 스페이스" 목록 + "모든 스페이스 보기".
   - **스페이스 안**: [스페이스 아이콘+이름] · 바로 가기 · **콘텐츠(페이지 트리 — 기존 PageTree + 제목검색 + 필터)** · 만들기 · 블로그(후속) · 캘린더(후속) · 스페이스 앱(후속).
3. **푸터**: Jira/팀(외부 ↗, 후속 생략 가능) · 더 보기.
- 접힘 상태는 localStorage 유지(기존 sidebarPrefs 재사용).

### 1.3 디렉토리 "모든 스페이스" = 테이블 (space 페이지.png)
- 상단: 자주 찾는 스페이스(아이콘+이름+별표 카드 한 줄) + "자세히 표시".
- 모든 스페이스: 제목 필터 input + 모든 분류 드롭다운 + 모든 필터 드롭다운(후속: 실제 필터 로직).
- **테이블 컬럼**: Space name(아이콘+이름, 클릭→이동) · Labels(칩, 데이터 없으면 빈칸) · Owner(아바타+이름 or "Not available") · Actions(눈/별표/… — 최소 별표).
- **Stage 3에서 카드로 바꾼 것을 테이블로 되돌린다.** (Space에 labels/owner 없으면 빈칸/"Not available".)

## 2. 아키텍처 (라우팅·컴포넌트)

- **`AppShell.tsx`(신규)** — 헤더(WikiTopBar, 토글 상시) + 접힘형 `GlobalSidebar` + `<Outlet/>`. 모든 라우트를 감싼다.
- **`GlobalSidebar.tsx`(신규)** — §1.2 3단. 컨텍스트 섹션은 현재 라우트로 판단(스페이스 안이면 `spaceId`로 PageTree, 아니면 별표목록).
- **`App.tsx`** — 최상위를 `AppShell`로 감싸고 그 안에 라우트 중첩:
  ```
  <Route element={<AppShell spaces=.../>}>
    <Route path="/home" element={<HomePage/>}/>
    <Route path="/spaces" element={<SpaceDirectoryPage/>}/>
    <Route path="/spaces/:spaceId" element={<SpaceOutlet/>}>  // 스페이스 컨텍스트 제공
      <Route index .../> <Route path="pages/*" .../>
    </Route>
  </Route>
  ```
- **`WikiLayout` 해체** — 사이드바는 GlobalSidebar로, 스페이스 컨텍스트(pages 로드·reloadPages)는 `SpaceOutlet`(스페이스 라우트 래퍼)로 이관. 화면 컴포넌트(PageView/Edit/Directory/Home)는 시그니처 유지.
- **`SpaceDirectoryPage`** — 카드 그리드 → 테이블(DS `Table` 재사용, 셀만 커스텀). 이미 있던 `space-directory-row-name`/`space-directory-star` CSS 재활용.
- 검색·만들기는 헤더(기존 WikiTopBar) 유지. 만들기 = 스페이스 안이면 새페이지/새스페이스, 밖이면 새 스페이스.

## 3. 단계 (SDD)

1. **디렉토리 테이블 복원** — SpaceDirectoryPage 카드→테이블 + 테스트 갱신(W7). (독립·빠름·즉시 체감)
2. **AppShell + GlobalSidebar 골격** — 헤더 토글 상시 + 글로벌 네비 + 컨텍스트 섹션(별표/트리). 라우트를 셸로 중첩.
3. **WikiLayout → SpaceOutlet 이관** — 스페이스 pages 로드·reloadPages를 셸 안 컨텍스트로. 기존 통합테스트(사이드바·트리·너비) 셸 기준 갱신.
4. **최근/스페이스 플라이아웃** — 글로벌 네비의 최근·스페이스 항목 플라이아웃(기존 SpaceFlyout 패턴 재사용).
5. (후속) 스페이스 개요 페이지, 앱그리드, 분류/필터 실동작, 블로그/캘린더.

각 단계 목업 테스트 green 유지. 화면 컴포넌트 공개 시그니처 최소 변경.

## 4. 리스크

- **통합테스트 대량 영향** — 셸 중첩·WikiLayout 해체로 `renderApp` 기반 테스트(사이드바/트리/토글/너비/디렉토리)가 구조 바뀜. 단계마다 해당 웨이브 테스트를 셸 기준으로 갱신(동작 보존, 셀렉터 조정).
- **기본 랜딩** — 이 셸이 서면 루트→/home 전환도 자연스러움(별도 결정이었던 것 흡수). renderApp 기본경로 의존 테스트 함께 갱신.
- 캡처의 앱그리드·Rovo·블로그·캘린더·분류필터는 **후속**(스펙에 명시, 무음 누락 금지).
